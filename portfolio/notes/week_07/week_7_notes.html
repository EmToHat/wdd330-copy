CH 11: Further Functions

  • In JavaScript, functions are first-class objects. This means they can be passed around in the same way as every other value.
  • Custom properties: Nothing to stop you from adding yr own properties to functions in the same way that you can add properties to any object in JavaScript. For example, you could add a description property to a function that describes what it does.
  • Memorization: If a function takes some time to compute a return value, we can save the result in a cache property. A useful feature of this is that it provides result caching, or memoization.
  • All functions have call() and apply() methods that can invoke a function with the value of this bound to an object that is provided as an argument.
  • Functions are able to dynamically redefine themselves in the body of the function, depending on certain conditions.

CH 13: AJAX

  • Asynchronous JavaScript and XML
  • AJAX is a method not a programming language, which is used for web app. 2 transmit & recieve data from the server at the same time w/o affecting the rest of the web page's content.
  • Helps to avoid a required page reload.
  • The goal of AJAX is to send small amnts of data 2 the server w/o the need to refresh the page.
  • XML stands for Extensible Markup Language.
  • XML is similar to HTML.
  • This markup language is used to encrypt messages to be read by humans and machines.
  • The web of computers or the internet can be separated into 2 parts: Clients and Servers.
  • A client, such as a web browser, requests a resource (a web page) from a server, then this request is proccessed and sends back a response to the client.
  • Resources are usually JSON data or small pieces of text or HTML.
  • A server, when requesting resources is required when using AJAX. This usually involves using a server-side lang. such as PHP, Ruby, Node.js or .NET which serves the data response following an AJAX request.
