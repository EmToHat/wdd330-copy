Programming Basics CH 02

Comments

  • Comments are extremely important: well-commented code makes it easier for anybody reading your code to understand what is happening in the code.
  • Two types of comments: single line and multi-line comments.

// this is a single line comment

/* this is a multi-line comment
anything in this space will be ignored
Multi-line comments are great for notes */

JS Grammar

  • JS syntax is similar to C programming language.
  • Two statements, one on each line.
  • Blocks don't need to be ended with a semicolon.

Reserved Words

These are words that are reserved for JS language and can't be used as a variable.

  • abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield
Important but not reserved.
  • undefined, NaN, Infinity

Primitive Data Types

  • JS has 7 different data types, 6 of them are primitive data types.
  • String, Symbol, Number, Boolean, Undefined, and Null.
  • Object isn't a primitive data type.
  • typeof special js operator for finding out the type of value.
  • Unary operator only requires one operand.
  • Binary operator two operands.
